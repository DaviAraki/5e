## Goal
Make any entity's detail view (spells + all entity pages) deep-linkable: copy a link, paste it elsewhere, and opening it shows the same entity.

## Approach
Back the shared `useMasterDetail` hook with a URL search param (`?s=...`) instead of plain `useState`. Selection then lives in the address bar for every page automatically. Add a small reusable copy-button to the detail panes.

## Changes

### 1. `src/hooks/useMasterDetail.ts` — make selection URL-backed (core change)
Rewrite so `selectedKey` is derived from `useSearchParams()`:
- Read: `selectedKey = searchParams.get("s")`.
- Write: `select(key)` and `setSelectedKey` update the `s` param via `setSearchParams(..., { replace: true })` (replace avoids spamming browser history). Clearing selection deletes the param.
- `isMobileDetail` stays local UI state, but initialize it `true` on mount when an `s` param is already present (so a pasted deep-link opens the detail pane on mobile).
- Preserve the existing returned API shape (`selectedKey`, `setSelectedKey`, `isMobileDetail`, `setIsMobileDetail`, `select`) so all 8 pages work unchanged.

Note: each page lives on its own route (`/spells`, `/bestiary`, …), so one generic `s` param is unambiguous and safe to reuse across pages.

### 2. `src/components/ShareLinkButton.tsx` — new reusable button
- Reads `window.location.href` at click time (already reflects `?s=…` after selection) → `navigator.clipboard.writeText`.
- try/catch fails silently, matching the existing pattern at `SpellBookPage.tsx:131-138`.
- Local `copied` state flips the label to "✓ Copied" for ~2s (no toast lib — codebase has none and uses only native dialogs).
- Styling reused from `SpellBookPage.tsx:273` (`rounded-md border border-border px-2 py-1 text-xs text-accent hover:bg-accent-subtle`).

### 3. Wire `<ShareLinkButton />` into detail components
- `SpellStatBlock.tsx` — add to the existing title/action row (lines 42-65), next to the "Add to Book" button. (Primary ask.)
- Add to the other entity detail components (Bestiary, Items, Feats, Classes, Backgrounds, Species, Books) for full coverage per your choice — confirm each exists during implementation and drop the button into its title area.

## Why this is minimal & safe
- **No new dependencies**, no router/route changes, no new `:id` routes.
- **Backwards compatible**: no `s` param → behaves exactly as today (empty detail placeholder).
- Single source of truth (URL) prevents state/URL drift.
- Deep-link load works even while data is still fetching: once `useSpells()` resolves, `selected` is derived from the param as before.
