# Add Grim Hollow 2024 (Full Mechanical Content)

## Context & Sources
- **Upstream data**: `TheGiddyLimit/homebrew` collection files (3 books, 2024/One edition):
  - `GrimHollowCG24` — Campaign Guide: 24 monsters, 118 items, 6 variantrules, 4 tables, 3 conditions, 20 deities, 1 disease
  - `GrimHollowPG24` — Player's Guide: 1 class, 40 subclasses, 25 backgrounds, 17 species, 42 feats, 101 spells, 61 items, 7 monsters, **Transformations (charoption) + 386 optionalfeatures**, 6 languages
  - `GrimHollowMG24` — Monster Grimoire: 409 monsters, 80 items, 5 legendary groups, 2 feats, 1 disease, 3 tables
- **Target**: this app's resolved JSON (`public/data/resolved/`). No merge script exists today; `scripts/` directory is absent. Schema is `{ entities, generatedAt, source, edition:"one" }`.
- **Skipped from upstream**: all `*Fluff` arrays (app keeps only `hasFluff`/`hasFluffImages` flags), `foundry*` arrays (Foundry-VTT only), item-system metadata (`itemProperty`/`itemType`/`itemMastery`/`itemTypeAdditionalEntries` — no home in app).

---

## Phase 1 — Merge tooling (reproducible import)

**New file:** `scripts/merge-grim-hollow.mjs` — zero-dependency Node 20+ ESM script.
- Fetches the 3 homebrew JSONs from `raw.githubusercontent.com/TheGiddyLimit/homebrew/master/collection/Ghostfire%20Gaming%3B%20Grim%20Hollow%20-%20*.json` (URL-encoded `;`). Supports `--local` flag to read vendored copies from `scripts/vendor/` instead.
- **Appends** to existing resolved files: `spells.json`, `bestiary.json`, `items.json` (item + itemGroup + magicvariant + baseitem), `classes.json`, `subclasses.json`, `backgrounds.json`, `species.json`, `feats.json`, `variantrules.json`, `books.json` (3 `book` objects with `group:"supplement-alt"`, `isOneEdition:true`).
- **Creates** new resolved files: `conditions.json`, `deities.json`, `diseases.json`, `languages.json`, `legendarygroups.json`, `tables.json`, `transformations.json` (charoption), `optionalfeatures.json`.
- **Writes** per-book readable content: `books/grimhollowcg24.json`, `books/grimhollowpg24.json`, `books/grimhollowmg24.json` (each = upstream `bookData[0].data` array).
- Refreshes `generatedAt` on every file it touches. Idempotent: strips any entity whose `source` starts with `GrimHollow` before re-appending (re-runnable without dupes).
- Validates output: each file is valid JSON, `entities` is an array, every entity has `name` + `source`.
- Prints a per-type count summary.

**New `package.json` script entry:** `"merge:gh": "node scripts/merge-grim-hollow.mjs"`.

## Notes / tradeoffs
- **`.mjs` for the merge script** (not `.ts`): zero new deps, runs on Node 20+ out of the box, avoids `tsx`/type-stripping friction for a build-time data tool. JSDoc-annotated.
- **No covers shipped**: GH entries get `cover.path: "covers/GrimHollow*.webp"`; images aren't bundled (consistent with all 63 existing books) so the `onError` text-initials fallback renders. Adding real `.webp` covers under `public/covers/` is optional and out of scope unless requested.
- **Grim Hollow's 2024 books set `isOneEdition:true`** → visible in the default "2024+" filter (consistent with the app's One-edition resolved set).
